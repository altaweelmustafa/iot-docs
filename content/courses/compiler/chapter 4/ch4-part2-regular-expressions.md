**Regular Languages** are a class of language important for lexical analysis ‚ÄĒ we use them to define and generate tokens. This class is defined recursively.

The three types of tokens (names, values, special symbols) are all generated by regular expressions. The scanner matches input strings against these patterns to recognize valid tokens.

---

## Defining a Language as a Set

**Alphabet V** ‚ÄĒ a set of symbols. For example: `V = {a, b, ..., z}`.

**String s** ‚ÄĒ a sequence of symbols taken from the alphabet. Examples over V: `abc`, `dsd`, `a`, `qwe`.

We define S as the set of all strings over some alphabet V. There are infinitely many strings over any alphabet.

A **concatenation** operation on S is defined as: given `x, y ‚ąą S`, then `X.Y = XY` is the string formed by following x with y. Note that `XY ‚Č† YX` ‚ÄĒ concatenation is **not commutative**.

The **empty string** őĽ satisfies `őĽX = XőĽ = X`. Lambda is the identity element of concatenation over S. Notation: `[S, ., őĽ]`.

**Language** ‚ÄĒ given an alphabet V, a language L over V is a set of strings formed from V. For example, given `V = {a, b, ..., z}`:

- `L1 = {aa, gb, c}`
- `L2 = {asdasd, qwe, asd}`
- `L3 = {abb}`

Each language Li is a subset of S (`Li ‚äÜ S`).

## Operations on Languages

Given alphabet V, let L = {set of all languages defined on V}, including `{}` (empty set) and `{őĽ}`.

### Concatenation "."

Given languages L and M over V:

`LM = {xy | x ‚ąą L, y ‚ąą M}`

**Example:** Given `L = {ab, bba}` and `M = {aa, bb}`:

- `LM = {abaa, abbb, bbaaa, bbabb}`
- `ML = {aaab, aabba, bbab, bbbba}`

Key properties:
- `LM ‚Č† ML` (not commutative)
- `L{őĽ} = {őĽ}L = L` (őĽ is the identity)
- `L{} = {}L = {}` (empty set annihilates)
- Concatenation is **closed** on L

### OR "|"

`L|M = {x | x ‚ąą L or x ‚ąą M} = L ‚ą™ M`

Key properties:
- `L|M = M|L` (commutative)
- `L|{} = {}|L = L` (empty set is identity for OR)

### Closure "*"

`L* = L‚Āį ‚ą™ L¬Ļ ‚ą™ L¬≤ ‚ą™ ... = ‚ą™(i=0 to ‚ąě) L‚ĀĪ`

`L+ = L* ‚ąí {őĽ}`

Where `L‚Āį = {őĽ}`, `L¬Ļ = L`, `L‚ĀŅ = L ¬∑ L ¬∑ ... ¬∑ L` (n times).

**Example:** Given `L = {ab, b}`:

- `L¬≤ = {abab, abb, bab, bb}`
- `L* = {őĽ, ab, b, abab, abb, bab, bb, ...}`
- `L+ = {ab, b, abab, abb, bab, bb, ...}`

## The Recursive Definition of Regular Languages

Given alphabet V, regular languages are defined recursively:

1. `‚ąÖ = {}` ‚ÄĒ the empty language is a regular language.
2. `őĽ = {őĽ}` ‚ÄĒ the empty string language is a regular language.
3. For every `a ‚ąą V`, `a = {a}` is a regular language (a singleton language).
4. Given R and S are regular languages, then:
   - `RS` is a regular language denoting `LR¬∑LS`
   - `R|S` is a regular language denoting `LR ‚ą™ LS`
   - `R*` is a regular language denoting `(LR)*`

**Examples** ‚ÄĒ given `R = {a}` and `S = {b}`:

- `RS = {ab}`
- `R|S = {a, b}`
- `R* = {őĽ, a, aa, aaa, ...}` ‚ÄĒ any number of a's
- `(RS)* = {őĽ, ab, abab, ababab, ...}` ‚ÄĒ any number of "ab"s
- `(a|b)* ` ‚ÄĒ any string of a's and b's, including őĽ
- `(0|1)*00` ‚ÄĒ any binary string ending with `00` (e.g. `100`, `000`, `1100`)
- `(a|b)*bbb(a|b)*` ‚ÄĒ any string containing at least 3 consecutive b's

## Defining Tokens Using Regular Languages

### Names

In programming languages, names are: a letter followed by letters or digits.

`Letter(Letter|digit)* = L(L|d)*`

### Values

**Integers:**

`[+|‚ąí]digit(digit)* = [+|‚ąí]dd* = [+|‚ąí]d+`

- `[x]` means take x zero or one time only.
- `d+` ‚ÄĒ one or more digits (no őĽ)
- `d*` ‚ÄĒ zero or more digits (includes őĽ)

**Floating Point Numbers:**

- Standard: `[+|‚ąí]d+.d+` (e.g. `0.6`, `123.456`, `-3.14`)
- Decimal: `[+|‚ąí]d*.d+` (e.g. `.6`, `1.123`, `-.5`)
- Scientific: `[+|‚ąí]d+.d+E[+|‚ąí]d+` (e.g. `1.5E+10`, `2.3E-5`)

### Special Symbols

Each special symbol has its own regular language:

- `+` = `{+}`
- `*` = `{*}`
- `<=` = `{<}{=}` = `{<=}`
- `++` = `{+}{+}` = `{++}`
