IoT System Components

Every IoT system — regardless of scale or complexity — is built from the same core set of components. Understanding these is the foundation for understanding how the deployment levels differ from each other.

Component Role
Device Hardware that performs sensing, actuation, identification, and remote monitoring
Resource Software on the device for reading sensors, processing data, controlling actuators, and enabling network access
Controller Service Native service on the device that relays data to web services and receives commands from the application; enables remote management
Database Local or cloud storage for data generated by the IoT device
Web Service Communication bridge between device, application, database, and analytics — implemented via REST or WebSocket
Analysis Component Processes IoT data and presents results in a form the user can understand
Application User-facing interface for monitoring, control, viewing system status, and processed data

The Six IoT Deployment Levels

IoT systems are classified into six levels based on three axes:

  • Number of nodes — single vs multiple
  • Where analysis happens — local vs cloud
  • Where data is stored — local vs cloud

As the level number increases, the system moves from a single self-contained local node toward a fully distributed, cloud-managed architecture with centralized intelligence.

Level Overview

Level Nodes Analysis Storage Application Example
1 Single Local Local Local Home Automation
2 Single Local Cloud Cloud Smart Irrigation
3 Single + gateway Cloud Cloud Cloud Package Tracking
4 Multiple Local + observer Cloud Cloud Noise Monitoring
5 Multiple end + coordinator Cloud Cloud Cloud Forest Fire Detection
6 Multiple independent Cloud (centralized controller) Cloud Cloud Smart Building
