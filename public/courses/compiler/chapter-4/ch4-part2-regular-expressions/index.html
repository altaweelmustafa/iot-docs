Regular Languages are a class of language important for lexical analysis — we use them to define and generate tokens. This class is defined recursively.

The three types of tokens (names, values, special symbols) are all generated by regular expressions. The scanner matches input strings against these patterns to recognize valid tokens.

Defining a Language as a Set

Alphabet V — a set of symbols. For example: V = {a, b, ..., z}.

String s — a sequence of symbols taken from the alphabet. Examples over V: abc, dsd, a, qwe.

We define S as the set of all strings over some alphabet V. There are infinitely many strings over any alphabet.

A concatenation operation on S is defined as: given x, y ∈ S, then X.Y = XY is the string formed by following x with y. Note that XY ≠ YX — concatenation is not commutative.

The empty string λ satisfies λX = Xλ = X. Lambda is the identity element of concatenation over S. Notation: [S, ., λ].

Language — given an alphabet V, a language L over V is a set of strings formed from V. For example, given V = {a, b, ..., z}:

  • L1 = {aa, gb, c}
  • L2 = {asdasd, qwe, asd}
  • L3 = {abb}

Each language Li is a subset of S (Li ⊆ S).

Operations on Languages

Given alphabet V, let L = {set of all languages defined on V}, including {} (empty set) and {λ}.

Concatenation “.”

Given languages L and M over V:

LM = {xy | x ∈ L, y ∈ M}

Example: Given L = {ab, bba} and M = {aa, bb}:

  • LM = {abaa, abbb, bbaaa, bbabb}
  • ML = {aaab, aabba, bbab, bbbba}

Key properties:

  • LM ≠ ML (not commutative)
  • L{λ} = {λ}L = L (λ is the identity)
  • L{} = {}L = {} (empty set annihilates)
  • Concatenation is closed on L

OR “|”

L|M = {x | x ∈ L or x ∈ M} = L ∪ M

Key properties:

  • L|M = M|L (commutative)
  • L|{} = {}|L = L (empty set is identity for OR)

Closure “*”

L* = L⁰ ∪ L¹ ∪ L² ∪ ... = ∪(i=0 to ∞) Lⁱ

L+ = L* − {λ}

Where L⁰ = {λ}, L¹ = L, Lⁿ = L · L · ... · L (n times).

Example: Given L = {ab, b}:

  • L² = {abab, abb, bab, bb}
  • L* = {λ, ab, b, abab, abb, bab, bb, ...}
  • L+ = {ab, b, abab, abb, bab, bb, ...}

The Recursive Definition of Regular Languages

Given alphabet V, regular languages are defined recursively:

  1. ∅ = {} — the empty language is a regular language.
  2. λ = {λ} — the empty string language is a regular language.
  3. For every a ∈ V, a = {a} is a regular language (a singleton language).
  4. Given R and S are regular languages, then:
    • RS is a regular language denoting LR·LS
    • R|S is a regular language denoting LR ∪ LS
    • R* is a regular language denoting (LR)*

Examples — given R = {a} and S = {b}:

  • RS = {ab}
  • R|S = {a, b}
  • R* = {λ, a, aa, aaa, ...} — any number of a’s
  • (RS)* = {λ, ab, abab, ababab, ...} — any number of “ab"s
  • (a|b)* — any string of a’s and b’s, including λ
  • (0|1)*00 — any binary string ending with 00 (e.g. 100, 000, 1100)
  • (a|b)*bbb(a|b)* — any string containing at least 3 consecutive b’s

Defining Tokens Using Regular Languages

Names

In programming languages, names are: a letter followed by letters or digits.

Letter(Letter|digit)* = L(L|d)*

Values

Integers:

[+|−]digit(digit)* = [+|−]dd* = [+|−]d+

  • [x] means take x zero or one time only.
  • d+ — one or more digits (no λ)
  • d* — zero or more digits (includes λ)

Floating Point Numbers:

  • Standard: [+|−]d+.d+ (e.g. 0.6, 123.456, -3.14)
  • Decimal: [+|−]d*.d+ (e.g. .6, 1.123, -.5)
  • Scientific: [+|−]d+.d+E[+|−]d+ (e.g. 1.5E+10, 2.3E-5)

Special Symbols

Each special symbol has its own regular language:

  • + = {+}
  • * = {*}
  • <= = {<}{=} = {<=}
  • ++ = {+}{+} = {++}
